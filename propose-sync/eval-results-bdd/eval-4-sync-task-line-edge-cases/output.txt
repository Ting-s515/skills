- 標記路徑以 `backend/` 開頭 → 掃描 `backend/docs/propose/`
- 標記路徑以 `docs/propose/` 開頭（無前綴）→ 掃描 `docs/propose/`

### Step 2：掃描 propose 目錄

**完成判斷規則：**

- 只判斷 `- [ ]`、`- [x]` 開頭的任務行；一般段落、標題、說明文字不納入判斷。
- 所有方括號標記皆採大小寫不敏感比對，例如 `[manual]`、`[MANUAL]`、`[bdd]`、`[BDD]` 視為對應相同標記。
- `[manual]` 任務不納入完成判斷；無論是否勾選，都不影響功能完成狀態。
- 非 `[manual]` 任務以 checkbox 作為主要完成狀態：
  - `- [x][bdd]` → 視為完成，代表非 UI 功能已完成 BDD 驗證。
  - `- [x][widget-test]` → 視為完成，代表 UI 功能已完成 Widget Test 驗證。
  - `- [x]` → 視為完成，適用於實作、文件更新、驗證等非測試追蹤任務。
  - `- [ ]`、`- [ ][bdd]`、`- [ ][widget-test]` 或任何未勾選的非 `[manual]` 任務 → 視為未完成。
- 有任何非 `[manual]` 任務未完成 → 該功能視為未完成。
- `03-tasks.md` 不存在 → 視為未完成
