Code Review: user-service.ts
=============================
日期：2026-05-17
審查對象：src/user-service.ts（staged 變更）

## 變更摘要

本次變更將 searchUsers 與 getUserById 兩個函式從空實作（stub）填入實際的資料庫查詢邏輯，使用 pg（node-postgres）風格的 pool.query 執行 SQL。

---

## 嚴重問題（必須修正）

### 1. SQL Injection — searchUsers（高風險）

```typescript
// 問題程式碼
const query = `SELECT id, name, email FROM users WHERE name LIKE '%${term}%' OR email LIKE '%${term}%'`;
const result = await pool.query(query);
```

**問題：** `term` 直接以字串插值注入 SQL，攻擊者可輸入如 `' OR '1'='1` 導致全表洩漏，甚至可執行 DROP TABLE 等破壞性指令。

**修正方式：** 改用參數化查詢（Parameterized Query）：

```typescript
const query = `SELECT id, name, email FROM users WHERE name LIKE $1 OR email LIKE $1`;
const result = await pool.query(query, [`%${term}%`]);
```

---

### 2. SQL Injection — getUserById（高風險）

```typescript
// 問題程式碼
const result = await pool.query(`SELECT * FROM users WHERE id = ${id}`);
```

**問題：** 雖然參數型別宣告為 `number`，但 TypeScript 型別系統無法保證執行時期（Runtime）安全。若呼叫端繞過型別傳入字串（如來自 HTTP query string），仍可造成 SQL Injection。

**修正方式：**

```typescript
const result = await pool.query('SELECT * FROM users WHERE id = $1', [id]);
```

---

## 中等問題（建議修正）

### 3. SELECT * 使用不當 — getUserById

```typescript
const result = await pool.query(`SELECT * FROM users WHERE id = ${id}`);
```

**問題：**
- 暴露密碼 hash、token 等敏感欄位給呼叫端
- 欄位異動時可能導致隱性破壞（silent breaking change）
- 無法從程式碼推斷回傳結構，降低可讀性

**建議：** 明確列出所需欄位：

```typescript
pool.query('SELECT id, name, email, created_at FROM users WHERE id = $1', [id])
```

---

### 4. 缺乏回傳型別宣告

兩個函式均缺少明確的回傳型別，TypeScript 無法提供靜態型別保護，呼叫端也無法得知預期結構。

**建議：** 定義 User 介面並宣告回傳型別：

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

export async function searchUsers(term: string): Promise<User[]> { ... }
export async function getUserById(id: number): Promise<User | null> { ... }
```

---

### 5. 缺乏錯誤處理

兩個函式均無 try/catch，任何資料庫連線失敗或查詢錯誤都會以未捕獲的 rejected Promise 往上傳播，可能導致伺服器 crash 或洩漏資料庫錯誤訊息給前端。

**建議：** 加入錯誤處理或至少在上層呼叫端統一處理，並考慮記錄 log：

```typescript
export async function getUserById(id: number): Promise<User | null> {
  try {
    const result = await pool.query('SELECT id, name, email FROM users WHERE id = $1', [id]);
    return result.rows[0] ?? null;
  } catch (err) {
    // 記錄 error log，避免洩漏原始 DB 錯誤
    throw new Error('Failed to fetch user');
  }
}
```
