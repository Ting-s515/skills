## 問題清單

------------------------------------------------------------
[CRITICAL] #1 — Props 使用 `any` 型別，違反規格技術規範

位置：src/components/TodoItem.tsx, 第 2 行

規格要求：
  「Props 必須使用明確的 TypeScript interface 定義，不可使用 `any`」

問題描述：
  實作中使用 `props: any`，完全放棄了 TypeScript 型別保護。
  這會導致以下風險：
  - 傳入錯誤型別的 props 不會在編譯期被發現
  - 使用端無法得知元件接受哪些 props
  - 重構時缺乏型別輔助，容易引入隱性錯誤

修正建議：
  定義明確的 interface：

  interface TodoItemProps {
    id: number;
    title: string;
    completed: boolean;
    priority: 'low' | 'medium' | 'high';
  }

  const TodoItem = ({ id, title, completed, priority }: TodoItemProps) => {
    // ...
  };

------------------------------------------------------------
[HIGH] #2 — JSX 中使用巢狀三元運算子，違反規格技術規範

位置：src/components/TodoItem.tsx, 第 16-18 行

規格要求：
  「狀態標籤邏輯必須易於閱讀與維護，避免巢狀三元運算子」

問題描述：
  return 中的 <span> 標籤直接使用了巢狀三元運算子：
    {props.completed ? '已完成' : props.priority === 'high' ? '緊急' : '進行中'}

  這正是規格明確禁止的寫法，可讀性低、維護困難。

修正建議：
  使用已計算好的 statusLabel 變數（程式碼中已定義但未使用），
  或改用 if/else 邏輯：

  // 方式一：使用 statusLabel 變數
  <span>{statusLabel}</span>

  // 方式二：使用 if/else 函式
  const getStatusLabel = () => {
    if (completed) return '已完成';
    if (priority === 'high') return '緊急';
    return '進行中';
  };

------------------------------------------------------------
[HIGH] #3 — statusLabel 變數計算後從未被使用（dead code）

位置：src/components/TodoItem.tsx, 第 3-7 行

問題描述：
  程式碼中計算了 statusLabel：
    const statusLabel = props.completed
      ? '已完成'
      : props.priority === 'high'
      ? '緊急'
      : '進行中';

  但 return 中並未使用 statusLabel，而是在 JSX 中重複了相同的巢狀三元邏輯。
  這造成：
  1. statusLabel 是無效的 dead code，每次 render 都多做一次無謂計算
  2. 邏輯重複定義兩次，未來若需修改，需同時改兩處，容易不一致

修正建議：
  刪除 JSX 中的巢狀三元，改用 statusLabel：
    <span>{statusLabel}</span>

  但同時需修正 #1（型別問題），確保 statusLabel 邏輯使用具型別的參數。

------------------------------------------------------------
[LOW] #4 — 元件未使用具名 export，export 方式與 placeholder 不一致

位置：src/components/TodoItem.tsx, 第 22 行

問題描述：
  舊版使用 `export const TodoItem = ...`（具名 export），
  新版改為 `export default TodoItem`（預設 export）。

  規格未明確規定 export 方式，但此改動會影響所有引用此元件的 import 語法，
  需確認是否為刻意設計，並與使用端保持一致。

修正建議：
  若專案慣例為具名 export，應改回：
    export const TodoItem = ...
  若確定改為 default export，需同步更新所有 import 端。
