# Web Search 強制原則

## 核心規則（不可違反）

**絕對禁止**直接用模型訓練資料回答以下類型問題，**必須先搜尋**：

| 類型 | 範例 |
|------|------|
| 時事 / 新聞 | 股市、政策、人事異動、科技新聞 |
| 套件 / 工具版本 | npm 版本、breaking change、API 變更 |
| 價格 / 數值 | 匯率、指數、商品價格 |
| 功能是否存在 | 某框架是否支援某功能 |
| 任何時效性資訊 | 含「最新」「目前」「現在」「今天」的問題 |

## 執行流程

### Step 1：判斷是否需要搜尋

以下任一條件成立，**立即進入搜尋流程**，不得跳過：

- 使用者 prompt 含有：最新、目前、現在、今天、近期、search、搜尋、查一下、查詢
- 問題涉及版本號、價格、匯率、指數、政策、法規
- 問題涉及特定工具、套件、框架的當前行為或支援狀況
- 任何「XX 現在怎樣了」、「XX 最新版本是什麼」類型問題

### Step 2：執行搜尋

1. 使用 **WebSearch** 搜尋關鍵字，優先搜尋官方來源
2. 若需要完整頁面內容，使用 **WebFetch** 取得原始資料
3. 搜尋至少 **1 次**，確認資料來源可信後再整理回答

### Step 3：組織回答

回答時必須標註：
- 資訊來源（URL 或來源名稱）
- 資料時效（搜尋日期或文章發布日期）

## 禁止行為

- 說「根據我的訓練資料...」後直接給答案，不搜尋
- 搜尋失敗後改用舊資料推論
- 認為「這個問題我應該知道」而略過搜尋

## 搜尋失敗時

若 WebSearch / WebFetch 無法取得資訊，必須明確告知：

> 「目前無法取得最新資訊，以下為我所知的歷史資料，可能已過時，請自行確認：」

然後才能提供舊資料作為參考，不可直接當作最新事實陳述。
