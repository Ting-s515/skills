# 功能流程：訂單通知信修正

## 功能概述

訂單狀態更新後，系統應正確觸發並寄送通知信給買家。

## 流程說明

1. 訂單狀態發生變更（例如：待出貨 → 已出貨、已出貨 → 已送達）
2. 系統觸發通知事件
   - IF 通知觸發條件滿足（狀態為需通知的類型）
     - 組裝通知信內容（訂單編號、新狀態、商品資訊）
     - 呼叫郵件服務寄送通知信至買家 Email
     - IF 寄送成功 → 記錄通知寄送 log
     - IF 寄送失敗 → 記錄錯誤 log，觸發重試機制（最多 3 次）
   - ELSE（狀態變更不需通知）
     - 跳過通知流程
3. 訂單狀態更新完成
