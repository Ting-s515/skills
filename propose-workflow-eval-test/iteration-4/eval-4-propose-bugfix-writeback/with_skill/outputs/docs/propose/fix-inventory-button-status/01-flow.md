# 功能流程：商品庫存按鈕狀態修正

## 功能概述

前台商品頁面應根據後端庫存數量動態更新「可購買」按鈕狀態，庫存歸零時禁用購買按鈕。

## 流程說明

1. 使用者進入商品頁面
2. 系統向後端請求商品資訊（含庫存數量）
3. 取得商品資料後判斷庫存狀態
   - IF 庫存數量 > 0
     - 顯示「加入購物車」按鈕（可點擊狀態）
   - ELSE（庫存數量 = 0）
     - 顯示「庫存不足」或禁用「加入購物車」按鈕（不可點擊狀態）
4. 使用者將商品加入購物車時
   - 系統再次確認庫存狀態
   - IF 庫存仍 > 0 → 加入購物車成功
   - ELSE → 顯示「商品已售完」錯誤提示，更新按鈕狀態為禁用
