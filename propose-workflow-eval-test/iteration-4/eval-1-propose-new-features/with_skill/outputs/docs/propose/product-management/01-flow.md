# 功能流程：商品管理

## 功能概述

管理員可透過後台新增商品（填寫名稱、價格、庫存、描述）、在商品列表依名稱篩選與依價格排序，並對每筆商品執行編輯或下架操作。

## 流程說明

### 1. 進入商品管理頁面

1. 管理員登入後台，點擊側邊欄「商品管理」
2. 系統載入商品列表（預設顯示所有商品，依建立時間降序排列）
3. 頁面顯示：商品列表（含名稱、價格、庫存數量、狀態欄位）與「新增商品」按鈕

### 2. 新增商品

1. 管理員點擊「新增商品」按鈕
2. 系統顯示新增商品表單（名稱、價格、庫存數量、商品描述）
3. 管理員填寫表單並送出
   - IF 所有必填欄位（名稱、價格、庫存數量）已填寫且格式正確 →
     系統呼叫 API 建立商品，關閉表單，商品列表新增該筆資料
   - IF 有必填欄位未填 → 顯示欄位驗證錯誤提示，不送出
   - IF 價格或庫存數量格式錯誤（非正數）→ 顯示格式錯誤提示，不送出
   - IF API 呼叫失敗 → 顯示錯誤提示，維持表單開啟狀態

### 3. 依名稱篩選商品

1. 管理員在篩選框輸入商品名稱關鍵字
2. 系統依輸入值篩選商品列表（部分比對，不區分大小寫）
   - IF 有符合商品 → 顯示篩選後列表
   - IF 無符合商品 → 顯示「查無符合商品」提示
3. 管理員清空篩選框 → 系統還原顯示所有商品列表

### 4. 依價格排序

1. 管理員點擊價格欄位標頭（或排序控制項）
2. 系統依價格重新排序列表
   - 第一次點擊 → 升序（低到高）
   - 第二次點擊 → 降序（高到低）
   - 第三次點擊 → 取消排序，還原預設排序

### 5. 編輯商品

1. 管理員點擊某商品的「編輯」按鈕
2. 系統顯示編輯表單，預填現有商品資料
3. 管理員修改欄位並送出
   - IF 表單驗證通過 → 系統呼叫 API 更新商品資料，關閉表單，列表即時反映變更
   - IF 表單驗證失敗 → 顯示對應欄位錯誤提示
   - IF API 呼叫失敗 → 顯示錯誤提示，維持表單開啟狀態

### 6. 下架商品

1. 管理員點擊某商品的「下架」按鈕
2. 系統顯示確認對話框：「確定要下架此商品嗎？」
3. 管理員確認
   - IF 確認 → 系統呼叫 API 執行下架，商品狀態更新為「已下架」，列表即時反映狀態
   - IF 取消 → 關閉對話框，不執行任何操作
4. 已下架商品在前台不顯示，但後台列表仍可見

### 邊界條件

- 價格必須為大於 0 的數字，庫存數量必須為大於等於 0 的整數
- 下架操作需有足夠的管理員權限，否則顯示「權限不足」錯誤
- 篩選與排序可同時套用（先篩選後排序）
- API 呼叫失敗時顯示錯誤提示，不更新前端狀態
