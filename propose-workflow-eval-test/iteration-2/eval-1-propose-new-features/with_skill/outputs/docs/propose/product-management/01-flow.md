# 功能流程：商品管理

## 功能概述

管理員在後台管理系統中新增商品、透過名稱篩選與價格排序瀏覽商品列表，以及對個別商品進行編輯或下架操作。

## 流程說明

### 比對條件

- 操作者身份：`role` 為管理員（admin）
- 操作對象：商品資料（`products` 資料表）

---

### 處理流程

#### 1. 新增商品

1. 管理員進入商品管理頁面，點擊「新增商品」
2. 管理員填寫商品表單，必填欄位：`name`（名稱）、`price`（價格）、`stock`（庫存數量）、`description`（商品描述）
3. 管理員送出新增表單
4. 系統驗證表單資料：
   - **驗證通過時：** 將商品資料寫入 `products` 資料表，回應成功，返回商品列表頁
   - **必填欄位為空時：** 顯示欄位驗證錯誤提示，不送出請求
   - **`price` 格式不合法（非正數）時：** 顯示欄位驗證錯誤提示
   - **`stock` 格式不合法（非零或正整數）時：** 顯示欄位驗證錯誤提示
   - **資料庫寫入失敗時：** 顯示系統錯誤提示，不關閉表單

---

#### 2. 查看商品列表（含篩選與排序）

1. 管理員進入商品管理頁面
2. 系統查詢 `products` 資料表，預設取得所有未下架商品
   - **有資料時：** 顯示商品列表（含名稱、價格、庫存數量、狀態等欄位）
   - **無資料時：** 顯示空列表提示
3. 管理員可輸入商品名稱關鍵字進行篩選：
   - 系統以 `name` 欄位模糊比對，回傳符合條件的商品列表
   - **篩選結果為空時：** 顯示「查無符合名稱的商品」提示
4. 管理員可選擇依價格排序：
   - **選擇「價格由低至高」：** 依 `price` 欄位 ASC 排序
   - **選擇「價格由高至低」：** 依 `price` 欄位 DESC 排序
   - 篩選與排序可同時作用，互不衝突

---

#### 3. 編輯商品

1. 管理員在列表中點擊某筆商品的「編輯」按鈕
2. 系統以該商品的 `id` 查詢 `products` 資料表：
   - **商品存在時：** 帶入現有資料，顯示編輯表單
   - **商品不存在時（id 無效）：** 返回列表頁並顯示錯誤提示
3. 管理員修改欄位後送出
4. 系統驗證與上述「新增商品」相同的規則
5. **驗證通過時：** 將更新資料寫入 `products` 資料表，回應成功，返回商品列表頁
6. **資料庫更新失敗時：** 顯示系統錯誤提示，保留表單現有內容

---

#### 4. 下架商品

1. 管理員在列表中點擊某筆商品的「下架」按鈕
2. 系統以該商品的 `id` 查詢 `products` 資料表：
   - **商品存在且狀態為「上架（active）」時：** 更新 `status` 為 `inactive`，回應成功，列表刷新
   - **商品已為「下架（inactive）」狀態時：** 不允許重複下架，顯示提示（Out of Scope：系統可選擇直接忽略或提示）
   - **商品不存在時（id 無效）：** 顯示錯誤提示
3. **資料庫更新失敗時：** 顯示系統錯誤提示，不更新狀態

---

### 邊界條件

| 情境 | 處理方式 |
|------|---------|
| 管理員未登入 | 導向登入頁，禁止存取 |
| 一般使用者嘗試存取管理後台 | 返回 403 無權限錯誤 |
| 新增/編輯必填欄位為空 | 顯示欄位驗證錯誤，不送出請求 |
| `price` 為負數或零 | 顯示欄位驗證錯誤，不送出請求 |
| `stock` 為負數 | 顯示欄位驗證錯誤，不送出請求 |
| 商品名稱搜尋輸入為空白字串 | 視同未篩選，回傳全部商品 |
| 商品不存在（id 無效） | 顯示錯誤提示並返回列表頁 |
| 資料庫操作失敗 | 顯示系統錯誤提示，狀態不變 |
