# 功能流程：使用者管理

## 功能概述

管理員可在後台查看與管理所有使用者帳號，包含列表瀏覽、搜尋、詳情查看及帳號狀態控制。

## 流程說明

### 比對條件

- 操作主體：已登入的管理員
- 資料來源：使用者資料表（users）

### 處理流程

#### 1. 查看使用者列表

1. 管理員進入使用者管理頁面
2. 系統查詢 `users` 資料表，取得所有使用者資料
3. **`email` 搜尋條件存在時：** 以 email 關鍵字過濾使用者清單（模糊比對）
4. **`email` 搜尋條件為空時：** 顯示全部使用者
5. 系統將結果以列表形式呈現
6. **查詢結果為空時：** 顯示「目前無使用者資料」提示，流程結束
7. **查詢結果不為空時：** 渲染使用者列表，等待管理員操作

#### 2. 查看使用者詳情

1. 管理員點擊列表中某筆使用者
2. 系統以該使用者 `id` 查詢 `users` 資料表
3. **使用者存在時：** 顯示詳情頁，包含欄位：`name`（姓名）、`email`、`created_at`（建立時間）、`order_count`（訂單數量）
4. **使用者不存在時（已被刪除或 id 無效）：** 回傳錯誤提示，導回列表頁，流程結束

#### 3. 停用 / 啟用使用者帳號

1. 管理員在列表頁或詳情頁點擊「停用」或「啟用」按鈕
2. 系統確認目前帳號 `status`：
   - **`status` 為啟用（active）時：** 將 `status` 更新為停用（inactive）
   - **`status` 為停用（inactive）時：** 將 `status` 更新為啟用（active）
3. **操作對象不存在時：** 回傳錯誤提示，不執行更新
4. **更新成功：** 列表/詳情頁反映最新狀態
5. **重複操作（已為目標狀態）：** 系統仍執行更新，結果不變（冪等行為）

### 邊界條件

- **未登入或無管理員權限：** 系統拒絕存取，導向登入頁，流程結束
- **搜尋字串過長（超過系統限制）：** 截斷或拒絕，顯示提示訊息（Out of Scope，待後續確認上限值）
- **訂單數量計算來源：** `order_count` 為關聯計算欄位，若訂單資料表無對應資料則顯示 0
