# Gherkin 驗收條件：商品管理

```gherkin
Feature: 商品管理
  管理員可在後台新增商品、瀏覽並篩選/排序商品列表，以及對既有商品進行編輯或下架操作

  Scenario: 管理員成功新增商品
    Given 管理員已登入後台
    When 管理員填寫有效的商品名稱、價格、庫存數量與描述並送出
    Then 系統建立該商品記錄
    And 導回商品列表頁

  Scenario: 新增商品時名稱為空
    Given 管理員已登入後台且在新增商品表單
    When 管理員未填寫名稱即送出表單
    Then 系統拒絕送出
    And 顯示名稱必填提示

  Scenario: 新增商品時價格格式錯誤
    Given 管理員已登入後台且在新增商品表單
    When 管理員輸入非正數或非數字的價格並送出
    Then 系統拒絕送出
    And 顯示價格格式錯誤提示

  Scenario: 新增商品時庫存數量格式錯誤
    Given 管理員已登入後台且在新增商品表單
    When 管理員輸入負數或非整數的庫存數量並送出
    Then 系統拒絕送出
    And 顯示庫存格式錯誤提示

  Scenario: 管理員查看商品列表
    Given 系統中存在若干商品資料
    And 管理員已登入後台
    When 管理員進入商品管理頁面
    Then 系統顯示所有商品的列表

  Scenario: 管理員以名稱篩選商品
    Given 系統中存在若干商品資料
    And 管理員已登入後台且在商品管理頁面
    When 管理員輸入商品名稱關鍵字並篩選
    Then 系統只顯示名稱符合關鍵字的商品

  Scenario: 管理員依價格排序商品
    Given 系統中存在若干商品資料
    And 管理員已登入後台且在商品管理頁面
    When 管理員選擇依價格排序
    Then 系統依指定方向（升冪或降冪）重新排列商品列表

  Scenario: 管理員成功編輯商品
    Given 系統中存在某商品
    And 管理員已登入後台
    When 管理員修改該商品的欄位並送出
    Then 系統更新該商品記錄

  Scenario: 編輯已不存在的商品
    Given 某商品已從系統中刪除
    And 管理員已登入後台
    When 管理員嘗試存取該商品的編輯頁
    Then 系統回傳錯誤提示
    And 導回商品列表頁

  Scenario: 管理員下架商品
    Given 某商品狀態為上架
    And 管理員已登入後台
    When 管理員對該商品執行下架操作
    Then 該商品狀態更新為下架

  Scenario: 未登入者嘗試存取商品管理
    Given 使用者未登入後台
    When 使用者嘗試存取商品管理頁面
    Then 系統拒絕存取
    And 導向登入頁
```
