# 驗收條件：商品管理

```gherkin
Feature: 商品管理

  Scenario: 管理員查看商品列表
    Given 管理員已登入後台
    When 管理員進入商品管理頁面
    Then 系統顯示所有商品列表，包含名稱、價格、庫存數量與上架狀態

  Scenario: 管理員依名稱篩選商品
    Given 管理員在商品管理頁面
    When 管理員輸入名稱關鍵字
    Then 系統顯示名稱包含該關鍵字的商品列表

  Scenario: 管理員依價格升冪排序
    Given 管理員在商品管理頁面
    When 管理員點擊價格欄位進行升冪排序
    Then 商品列表依價格由低至高排列

  Scenario: 管理員新增商品（成功）
    Given 管理員在商品管理頁面
    When 管理員點擊「新增商品」，填寫名稱、價格、庫存數量後送出
    Then 系統建立新商品，返回列表頁並顯示成功訊息

  Scenario: 管理員新增商品（欄位驗證失敗）
    Given 管理員在新增商品表單
    When 管理員未填寫必填欄位或輸入非法價格後點擊「儲存」
    Then 系統顯示各欄位錯誤提示，不送出請求

  Scenario: 管理員編輯商品
    Given 管理員在商品管理列表頁面
    When 管理員點擊某商品的「編輯」，修改欄位後點擊「儲存」
    Then 系統更新商品資料並顯示成功訊息

  Scenario: 管理員下架商品
    Given 管理員在商品管理列表頁面，該商品目前為上架狀態
    When 管理員點擊「下架」並在確認對話框中確認
    Then 系統將商品狀態更新為下架，列表狀態即時更新

  Scenario: 管理員取消下架操作
    Given 管理員點擊「下架」，確認對話框已開啟
    When 管理員點擊「取消」
    Then 系統關閉對話框，商品狀態不變
```
