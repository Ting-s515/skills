# 驗收條件：庫存歸零時購買按鈕狀態修正

```gherkin
Feature: 商品庫存狀態與購買按鈕連動

  Scenario: 庫存充足時購買按鈕為啟用狀態
    Given 商品庫存數量為 5
    When 使用者進入該商品頁面
    Then 購買按鈕應顯示為啟用狀態（可點擊）

  Scenario: 庫存歸零時購買按鈕為禁用狀態
    Given 商品庫存數量為 0
    When 使用者進入該商品頁面
    Then 購買按鈕應顯示為禁用狀態（不可點擊）
    And 按鈕文字應顯示「已售完」或「補貨中」

  Scenario: 庫存補貨後購買按鈕恢復啟用
    Given 商品庫存數量原本為 0，按鈕顯示禁用
    When 後台補貨使庫存數量變為 10
    And 前台取得最新商品資料
    Then 購買按鈕應恢復啟用狀態

  Scenario: 後端 API 未回傳庫存欄位時，購買按鈕預設禁用
    Given 後端 API 回傳商品資料但不含庫存欄位
    When 使用者進入該商品頁面
    Then 購買按鈕應顯示為禁用狀態，避免誤購
```
