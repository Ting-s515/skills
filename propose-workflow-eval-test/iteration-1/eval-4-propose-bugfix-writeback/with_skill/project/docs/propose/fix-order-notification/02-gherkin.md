# 驗收條件：訂單狀態更新通知修正

```gherkin
Feature: 訂單狀態更新通知

  Scenario: 訂單狀態更新為需通知類型時，買家收到通知信
    Given 系統中存在一筆訂單，狀態為「已付款」
    And 買家具有有效的註冊信箱
    When 管理員將訂單狀態更新為「已出貨」
    Then 系統應寄送通知信至買家信箱
    And 通知信內容應包含訂單編號與新狀態

  Scenario: 訂單狀態更新為不需通知類型時，不寄出通知信
    Given 系統中存在一筆訂單，狀態為「處理中」
    When 管理員將訂單狀態更新為另一個內部流轉狀態
    Then 系統不應寄送任何通知信
    And 系統應記錄日誌說明無需通知

  Scenario: 買家信箱無效時，系統記錄錯誤但不中斷訂單流程
    Given 系統中存在一筆訂單，買家信箱格式無效
    When 管理員將訂單狀態更新為「已完成」
    Then 系統應記錄發信錯誤
    And 訂單狀態仍應成功更新

  Scenario: 郵件服務暫時不可用時，進入重試佇列
    Given 郵件服務目前不可用
    And 系統中存在一筆訂單需要發送通知
    When 管理員更新訂單狀態觸發通知
    Then 系統應將通知任務加入重試佇列
    And 重試次數不超過 3 次
```
