```gherkin
Feature: 商品管理
  管理員在後台管理系統中新增商品、透過名稱篩選與價格排序瀏覽商品列表，以及對個別商品進行編輯或下架操作

  Scenario: 成功新增商品
    Given 管理員已登入後台
    And 管理員填寫完整的商品名稱、價格、庫存數量與商品描述
    When 管理員送出新增商品表單
    Then 商品資料寫入系統
    And 返回商品列表頁面

  Scenario: 新增商品時必填欄位為空
    Given 管理員已進入新增商品表單
    And 管理員未填寫一或多個必填欄位
    When 管理員送出新增商品表單
    Then 顯示欄位驗證錯誤提示
    And 不建立新商品資料

  Scenario: 新增商品時價格格式不合法
    Given 管理員已進入新增商品表單
    And 管理員輸入的價格為非正數
    When 管理員送出新增商品表單
    Then 顯示價格欄位驗證錯誤提示
    And 不建立新商品資料

  Scenario: 新增商品時庫存格式不合法
    Given 管理員已進入新增商品表單
    And 管理員輸入的庫存數量為負數
    When 管理員送出新增商品表單
    Then 顯示庫存欄位驗證錯誤提示
    And 不建立新商品資料

  Scenario: 查看商品列表
    Given 管理員已登入後台
    And 系統中存在若干上架商品
    When 管理員進入商品管理頁面
    Then 顯示所有上架商品列表

  Scenario: 依名稱篩選商品
    Given 管理員已進入商品管理頁面
    When 管理員輸入商品名稱關鍵字進行篩選
    Then 顯示名稱符合關鍵字的商品列表

  Scenario: 名稱篩選結果為空
    Given 管理員已進入商品管理頁面
    When 管理員輸入不存在的商品名稱關鍵字進行篩選
    Then 顯示查無符合名稱的商品提示

  Scenario: 依價格由低至高排序
    Given 管理員已進入商品管理頁面
    When 管理員選擇「價格由低至高」排序
    Then 商品列表依價格由低至高顯示

  Scenario: 依價格由高至低排序
    Given 管理員已進入商品管理頁面
    When 管理員選擇「價格由高至低」排序
    Then 商品列表依價格由高至低顯示

  Scenario: 成功編輯商品
    Given 管理員已進入某筆商品的編輯表單
    And 管理員修改了商品的一或多個欄位
    When 管理員送出編輯表單
    Then 商品資料更新至系統
    And 返回商品列表頁面

  Scenario: 編輯不存在的商品
    Given 管理員嘗試存取不存在的商品 id 進行編輯
    When 系統查詢該商品
    Then 返回商品列表頁面
    And 顯示錯誤提示

  Scenario: 下架上架中的商品
    Given 管理員已查看某筆狀態為「上架」的商品
    When 管理員執行下架操作
    Then 該商品狀態更新為「下架」

  Scenario: 資料庫操作失敗
    Given 管理員嘗試新增、編輯或下架商品
    When 資料庫操作失敗
    Then 商品資料不變
    And 顯示系統錯誤提示

  Scenario: 無管理員權限的使用者嘗試存取後台
    Given 使用者已登入但身份非管理員
    When 使用者嘗試進入商品管理頁面
    Then 返回 403 無權限錯誤
```
