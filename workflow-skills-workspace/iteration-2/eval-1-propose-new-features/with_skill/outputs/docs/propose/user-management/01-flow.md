# 功能流程：使用者管理

## 功能概述

管理員在後台管理系統中查看、搜尋使用者列表，查看個別使用者詳情，以及停用或啟用使用者帳號。

## 流程說明

### 比對條件

- 操作者身份：`role` 為管理員（admin）
- 操作對象：使用者帳號（`users` 資料表）

---

### 處理流程

#### 1. 查看使用者列表

1. 管理員進入使用者管理頁面
2. 系統查詢 `users` 資料表，取得所有使用者資料
   - **有資料時：** 顯示使用者列表，包含 email、帳號狀態等欄位
   - **無資料時：** 顯示空列表提示（Out of Scope：無需特殊錯誤處理）
3. 管理員可在搜尋框輸入 email 關鍵字進行搜尋
   - **已輸入搜尋關鍵字時：** 系統以 `email` 欄位模糊比對，回傳符合條件的使用者列表
   - **搜尋結果為空時：** 顯示「查無符合條件的使用者」提示
   - **未輸入時：** 顯示全部使用者

---

#### 2. 查看使用者詳情

1. 管理員在列表中點擊某筆使用者
2. 系統以該使用者的 `id` 查詢 `users` 資料表
   - **使用者存在時：** 顯示詳情頁，包含：姓名、`email`、建立時間（`created_at`）、訂單數量（關聯 `orders` 資料表計算）
   - **使用者不存在時（id 無效）：** 返回列表頁並顯示錯誤提示

---

#### 3. 停用 / 啟用使用者帳號

1. 管理員在使用者詳情頁或列表中操作停用 / 啟用按鈕
2. 系統讀取該使用者目前的 `status` 欄位
3. **當前狀態為「啟用（active）」時：**
   - 系統將 `status` 更新為 `inactive`
   - 回應成功，列表或詳情頁刷新狀態顯示
4. **當前狀態為「停用（inactive）」時：**
   - 系統將 `status` 更新為 `active`
   - 回應成功，列表或詳情頁刷新狀態顯示
5. **操作失敗時（如資料庫錯誤）：** 顯示錯誤提示，不更新狀態
6. **重複操作（連續點擊）：** 系統以最後一次操作為準（無冪等額外保護，Out of Scope）

---

### 邊界條件

| 情境 | 處理方式 |
|------|---------|
| 管理員未登入 | 導向登入頁，禁止存取 |
| 一般使用者嘗試存取管理後台 | 返回 403 無權限錯誤 |
| email 搜尋輸入為空白字串 | 視同未輸入，回傳全部使用者 |
| 使用者不存在（id 無效） | 返回列表頁並顯示錯誤提示 |
| 停用/啟用操作資料庫失敗 | 顯示錯誤提示，不更新狀態 |
