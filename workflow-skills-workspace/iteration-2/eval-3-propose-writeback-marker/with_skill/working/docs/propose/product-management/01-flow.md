# 功能流程：商品管理

## 功能概述

管理員可在後台新增商品、瀏覽並篩選/排序商品列表，以及對既有商品進行編輯或下架操作。

## 流程說明

### 比對條件

- 操作主體：已登入的管理員
- 資料來源：商品資料表（products）

### 處理流程

#### 1. 新增商品

1. 管理員在商品管理頁面點擊「新增商品」
2. 系統呈現新增表單，包含欄位：`name`（名稱）、`price`（價格）、`stock`（庫存數量）、`description`（商品描述）
3. 管理員填寫並送出表單
4. 系統驗證輸入：
   - **`name` 為空時：** 拒絕送出，顯示必填提示
   - **`price` 為非正數或非數字時：** 拒絕送出，顯示格式錯誤提示
   - **`stock` 為負數或非整數時：** 拒絕送出，顯示格式錯誤提示
5. **驗證通過：** 寫入 `products` 資料表，新增成功後導回商品列表頁
6. **名稱重複（若有唯一性限制）：** 回傳衝突錯誤提示（Out of Scope，待確認是否有唯一性限制）

#### 2. 查看商品列表（含篩選與排序）

1. 管理員進入商品管理頁面
2. 系統查詢 `products` 資料表，取得所有商品資料
3. **`name` 篩選條件存在時：** 以名稱關鍵字過濾商品清單（模糊比對）
4. **`name` 篩選條件為空時：** 顯示全部商品
5. **價格排序條件存在時：** 依 `price` 欄位排序（升冪或降冪）
6. **價格排序條件未指定時：** 使用預設排序（依 `id` 升冪）
7. **查詢結果為空時：** 顯示「目前無商品資料」提示，流程結束
8. **查詢結果不為空時：** 渲染商品列表，等待管理員操作

#### 3. 編輯商品

1. 管理員在商品列表點擊「編輯」
2. 系統以 `id` 查詢 `products` 資料表
3. **商品存在時：** 顯示預填表單，欄位同新增商品
4. **商品不存在時：** 回傳錯誤提示，導回列表頁，流程結束
5. 管理員修改欄位後送出
6. 系統執行與新增相同的輸入驗證
7. **驗證通過：** 更新 `products` 資料表對應記錄，返回商品列表頁

#### 4. 下架商品

1. 管理員在列表或編輯頁點擊「下架」
2. 系統以 `id` 查詢 `products` 資料表
3. **商品存在且狀態為上架（active）：** 將 `status` 更新為下架（inactive）
4. **商品已下架：** 系統仍執行更新，結果不變（冪等行為）
5. **商品不存在時：** 回傳錯誤提示，不執行更新

### 邊界條件

- **未登入或無管理員權限：** 系統拒絕存取，導向登入頁，流程結束
- **`description` 欄位：** 允許為空，無強制必填（Out of Scope，待確認最大字元限制）
- **同時篩選名稱與排序價格：** 兩個條件可同時套用，互不排斥
