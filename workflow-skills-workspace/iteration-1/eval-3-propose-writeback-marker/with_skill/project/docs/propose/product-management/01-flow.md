# 功能流程：商品管理

## 功能概述

管理員可在後台新增商品、透過篩選與排序瀏覽商品列表，並對既有商品進行編輯或下架操作。

## 流程說明

### 主流程：查看商品列表

1. 管理員進入商品管理頁面
2. 系統載入所有商品列表
3. 管理員可使用以下操作過濾與排序：
   - 依名稱關鍵字篩選：輸入關鍵字後即時過濾列表
   - 依價格排序：點擊價格欄位切換升冪/降冪排序
4. 列表顯示商品名稱、價格、庫存數量、上架狀態

### 子流程：新增商品

1. 管理員點擊「新增商品」按鈕
2. 系統顯示新增表單，包含欄位：
   - 名稱（必填）
   - 價格（必填，需為正整數）
   - 庫存數量（必填，需為非負整數）
   - 商品描述（選填）
3. 管理員填寫後點擊「儲存」
   - 驗證通過：系統建立商品，返回列表頁並顯示成功訊息
   - 驗證失敗：顯示各欄位錯誤提示，不送出請求

### 子流程：編輯商品

1. 管理員在列表中點擊某商品的「編輯」
2. 系統顯示編輯表單，預填現有資料
3. 管理員修改欄位後點擊「儲存」
   - 驗證通過：系統更新商品資料，顯示成功訊息
   - 驗證失敗：顯示錯誤提示

### 子流程：下架商品

1. 管理員在列表中點擊某商品的「下架」
2. 系統顯示確認對話框
3. 管理員確認
   - 確認：系統將商品狀態設為下架，列表狀態即時更新
   - 取消：關閉對話框，不做任何變更

### 邊界條件

- 已下架商品不在前台顯示，但在後台仍可查看與編輯
- 價格不可為 0 或負數
- 庫存數量可為 0（表示暫時缺貨，但仍上架）
