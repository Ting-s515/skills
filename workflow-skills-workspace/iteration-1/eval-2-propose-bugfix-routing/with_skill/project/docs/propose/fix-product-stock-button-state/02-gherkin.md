# 驗收條件：修正商品庫存歸零時購買按鈕狀態

```gherkin
Feature: 商品庫存歸零時購買按鈕狀態

  Scenario: 商品有庫存時，購買按鈕顯示為可購買狀態
    Given 一件商品的庫存數量為 5
    When 使用者進入該商品的前台頁面
    Then 購買按鈕應顯示為啟用狀態
    And 按鈕文字應為「加入購物車」或「立即購買」

  Scenario: 商品庫存歸零時，購買按鈕顯示為不可購買狀態
    Given 一件商品的庫存數量為 0
    When 使用者進入該商品的前台頁面
    Then 購買按鈕應顯示為停用狀態（disabled）
    And 按鈕文字應為「已售完」
    And 使用者點擊按鈕時，不觸發任何購買動作

  Scenario: 庫存即時歸零時，按鈕狀態動態更新為不可購買
    Given 使用者正在瀏覽一件有庫存的商品頁面
    When 系統接收到該商品庫存歸零的更新事件
    Then 購買按鈕應立即切換為停用狀態
    And 按鈕文字應更新為「已售完」
    And 不需整頁重整

  Scenario: 商品庫存由零恢復補貨時，按鈕狀態動態更新為可購買
    Given 使用者正在瀏覽一件已售完的商品頁面
    When 系統接收到該商品庫存補貨（stock_quantity > 0）的更新事件
    Then 購買按鈕應立即切換為啟用狀態
    And 按鈕文字應更新為「加入購物車」

  Scenario: stock_quantity 為 NULL 時，購買按鈕顯示為不可購買狀態
    Given 一件商品的 stock_quantity 欄位值為 NULL
    When 使用者進入該商品的前台頁面
    Then 購買按鈕應顯示為停用狀態
    And 按鈕文字應為「已售完」

  Scenario: 前端判斷失效時，後端 API 阻擋無庫存商品的購買請求
    Given 一件商品的庫存數量為 0
    And 前端按鈕停用判斷因異常未生效
    When 使用者對該商品送出購買請求
    Then 後端 API 應回傳庫存不足錯誤
    And 前台應顯示提示訊息「此商品已售完，無法購買」
```
