# 驗收條件：商品管理

```gherkin
Feature: 商品管理

  Scenario: 管理員查看商品列表
    Given 管理員已登入後台
    When 管理員進入「商品管理」頁面
    Then 系統顯示所有商品的分頁列表

  Scenario: 管理員依名稱篩選商品
    Given 管理員在商品列表頁面
    When 管理員在搜尋欄輸入商品名稱關鍵字
    Then 系統顯示名稱符合關鍵字的商品列表

  Scenario: 篩選結果為空
    Given 管理員在商品列表頁面
    When 管理員輸入不存在的商品名稱
    Then 系統顯示「無符合條件的商品」提示

  Scenario: 管理員依價格由低到高排序商品
    Given 管理員在商品列表頁面
    When 管理員選擇「價格由低到高」排序
    Then 系統以價格升序重新排列商品列表

  Scenario: 管理員依價格由高到低排序商品
    Given 管理員在商品列表頁面
    When 管理員選擇「價格由高到低」排序
    Then 系統以價格降序重新排列商品列表

  Scenario: 管理員成功新增商品
    Given 管理員在商品列表頁面
    When 管理員點擊「新增商品」並填寫名稱、價格、庫存數量、商品描述後點擊「儲存」
    Then 系統建立該商品並顯示成功訊息
    And 新商品出現在商品列表中

  Scenario: 新增商品時必填欄位為空
    Given 管理員開啟新增商品表單
    When 管理員未填寫必填欄位並點擊「儲存」
    Then 系統顯示對應欄位的錯誤提示，不送出表單

  Scenario: 新增商品時價格輸入非正數
    Given 管理員開啟新增商品表單
    When 管理員在價格欄位輸入負數或零
    Then 系統顯示「價格必須為正數」錯誤提示，不送出表單

  Scenario: 新增商品 API 失敗
    Given 管理員填寫完整的新增商品表單並點擊「儲存」
    When API 回傳錯誤
    Then 系統顯示錯誤訊息，表單維持開啟狀態

  Scenario: 管理員成功編輯商品
    Given 管理員在商品列表頁面
    When 管理員點擊「編輯」，修改商品資料後點擊「儲存」
    Then 系統更新商品資料並顯示成功訊息
    And 列表中該商品資料即時更新

  Scenario: 管理員成功下架商品
    Given 管理員在商品列表頁面，且目標商品為上架狀態
    When 管理員點擊「下架」按鈕並確認操作
    Then 系統將商品標記為下架並顯示成功訊息
    And 列表中該商品狀態即時更新

  Scenario: 管理員取消下架確認對話框
    Given 管理員點擊「下架」按鈕後系統顯示確認對話框
    When 管理員點擊「取消」
    Then 對話框關閉，商品狀態維持不變

  Scenario: 下架 API 呼叫失敗
    Given 管理員確認下架操作
    When API 回傳錯誤
    Then 系統顯示錯誤訊息，商品狀態維持不變
```
