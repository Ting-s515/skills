# 功能流程：商品管理

## 功能概述

管理員可在後台新增商品、透過名稱篩選與價格排序瀏覽商品列表，並對個別商品執行編輯或下架操作。

## 流程說明

### 1. 商品列表（含篩選與排序）

**前置條件：** 管理員已登入後台

**主要流程：**
1. 管理員進入「商品管理」頁面
2. 系統載入並顯示所有商品列表（分頁顯示）
3. 管理員可在搜尋欄輸入商品名稱關鍵字進行篩選
4. 管理員可選擇排序方式（價格由低到高 / 由高到低）
5. 系統依篩選條件與排序設定更新列表

**判斷分支：**
- 篩選結果有資料 → 顯示符合條件的商品列表
- 篩選結果無資料 → 顯示「無符合條件的商品」提示

### 2. 新增商品

**前置條件：** 管理員在商品列表頁面

**主要流程：**
1. 管理員點擊「新增商品」按鈕
2. 系統開啟新增商品表單（名稱、價格、庫存數量、商品描述）
3. 管理員填寫所有欄位並點擊「儲存」
4. 系統驗證表單資料
5. 驗證通過 → API 建立商品，回饋成功訊息，列表新增該商品

**判斷分支：**
- 必填欄位未填 → 顯示欄位錯誤提示，不送出
- 價格或庫存輸入非正數 → 顯示格式錯誤提示，不送出
- API 建立失敗 → 顯示錯誤訊息，表單維持開啟狀態

### 3. 編輯商品

**前置條件：** 管理員在商品列表頁面

**主要流程：**
1. 管理員點擊某筆商品的「編輯」按鈕
2. 系統開啟編輯表單，預填現有資料（名稱、價格、庫存數量、商品描述）
3. 管理員修改欄位並點擊「儲存」
4. 系統驗證並呼叫 API 更新商品
5. 成功後關閉表單，列表資料即時更新

**判斷分支（同新增商品的驗證邏輯）**

### 4. 下架商品

**前置條件：** 管理員在商品列表頁面

**主要流程：**
1. 管理員點擊某筆商品的「下架」按鈕
2. 系統顯示確認對話框
3. 管理員確認操作
4. 系統呼叫 API 將商品標記為下架，回饋成功訊息
5. 列表中該商品狀態即時更新（或從列表移除，依業務規則而定）

**判斷分支：**
- 管理員取消確認 → 關閉對話框，不執行操作
- API 呼叫失敗 → 顯示錯誤訊息，商品狀態維持不變
