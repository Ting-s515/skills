# 功能流程：使用者管理

## 功能概述

管理員可在後台查看、搜尋使用者列表，檢視個別使用者詳情，並對帳號執行停用或啟用操作。

## 流程說明

### 1. 使用者列表

**前置條件：** 管理員已登入後台

**主要流程：**
1. 管理員進入「使用者管理」頁面
2. 系統載入並顯示所有使用者列表（分頁顯示）
3. 管理員可在搜尋欄輸入 email 關鍵字
4. 系統依 email 即時篩選並更新列表

**判斷分支：**
- 搜尋結果有資料 → 顯示符合條件的使用者列表
- 搜尋結果無資料 → 顯示「無符合條件的使用者」提示

### 2. 查看使用者詳情

**前置條件：** 管理員在使用者列表頁面

**主要流程：**
1. 管理員點擊某筆使用者列表項目
2. 系統導向（或開啟 Modal）顯示該使用者詳情
3. 詳情內容包含：姓名、email、建立時間、訂單數量

### 3. 停用 / 啟用使用者帳號

**前置條件：** 管理員在使用者列表或詳情頁面

**主要流程：**
1. 管理員點擊「停用」或「啟用」按鈕
2. 系統顯示確認對話框（避免誤操作）
3. 管理員確認操作
4. 系統更新帳號狀態並回饋成功訊息
5. 列表中該使用者的狀態標籤即時更新

**判斷分支：**
- 管理員取消確認 → 關閉對話框，不執行操作
- API 呼叫失敗 → 顯示錯誤訊息，帳號狀態維持不變

**邊界條件：**
- 停用目標為當前登入管理員自身帳號 → 系統禁止操作並提示錯誤
