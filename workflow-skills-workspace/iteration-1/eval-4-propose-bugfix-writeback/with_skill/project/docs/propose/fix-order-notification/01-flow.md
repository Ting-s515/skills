# 功能流程：訂單狀態更新通知修正

## 功能概述

確保訂單狀態發生變更時，系統能正確觸發通知流程並寄送通知信給買家。

## 流程說明

### 正常流程

1. 管理員在後台更新訂單狀態（例如：待付款 → 已付款、已付款 → 出貨中）
2. 系統儲存新狀態至資料庫
3. 系統檢查該狀態變更是否屬於「需通知」的類型
4. 若需通知，系統組裝通知信內容（訂單編號、新狀態、買家姓名等）
5. 系統透過郵件服務發送通知信至買家的註冊信箱
6. 系統記錄通知發送結果（成功 / 失敗）

### 判斷分支

- **狀態變更屬於通知清單**（如：已付款、已出貨、已完成、已取消）→ 觸發發信
- **狀態變更不在通知清單**（如：系統內部狀態流轉）→ 不發信，僅記錄日誌

### 邊界條件

- 買家信箱不存在或格式無效 → 記錄錯誤，不中斷訂單流程
- 郵件服務暫時不可用 → 進入重試佇列，最多重試 3 次
- 同一訂單短時間內連續狀態變更 → 每次狀態變更各自觸發獨立通知，不合併
