# 功能流程：庫存歸零時購買按鈕狀態修正

## 功能概述

確保商品庫存歸零時，前台購買按鈕正確反映庫存狀態，顯示為不可購買。

## 流程說明

### 正常流程（庫存充足）

1. 使用者進入商品頁面
2. 系統向後端查詢商品資訊，包含庫存數量
3. 庫存數量 > 0 → 購買按鈕顯示為啟用狀態（可點擊）

### 正常流程（庫存歸零）

1. 使用者進入商品頁面
2. 系統向後端查詢商品資訊，庫存數量為 0
3. 購買按鈕應顯示為禁用狀態（不可點擊），顯示「已售完」或「補貨中」文字

### 即時更新流程

1. 使用者正在商品頁面瀏覽
2. 後台管理員操作導致庫存歸零（其他買家購買完畢）
3. 前台應在下次資料刷新或 API 回應時更新按鈕狀態

### 邊界條件

- 庫存從有庫存變為 0 → 按鈕立即變為禁用
- 庫存從 0 補貨後 > 0 → 按鈕恢復啟用
- 後端 API 回傳庫存欄位缺失 → 按鈕預設為禁用（防止誤購）
