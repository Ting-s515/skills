# 驗收條件：訂單通知信修正

## Feature: 訂單狀態變更通知信

### Scenario: 訂單狀態更新為需通知類型時成功寄送通知信
```
Given 訂單編號 #12345 目前狀態為「待出貨」
When 管理員將訂單狀態更新為「已出貨」
Then 系統應寄送通知信至買家 Email
And 通知信應包含訂單編號 #12345 與最新狀態「已出貨」
And 系統應記錄一筆寄送成功的 log
```

### Scenario: 寄送通知信失敗時觸發重試機制
```
Given 訂單狀態已更新為「已出貨」
And 郵件服務暫時不可用
When 系統嘗試寄送通知信
Then 系統應記錄錯誤 log
And 系統應在重試間隔後自動重試（最多 3 次）
```

### Scenario: 訂單狀態變更為不需通知類型時跳過通知
```
Given 訂單狀態為內部備註更新（不需通知買家）
When 管理員更新訂單備註
Then 系統不應寄送通知信給買家
```
