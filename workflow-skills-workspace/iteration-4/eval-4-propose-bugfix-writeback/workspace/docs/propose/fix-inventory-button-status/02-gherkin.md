# 驗收條件：商品庫存按鈕狀態修正

## Feature: 商品頁面庫存狀態動態顯示

### Scenario: 商品有庫存時顯示可點擊的購買按鈕
```
Given 商品 A 的庫存數量為 5
When 使用者進入商品 A 的頁面
Then 頁面應顯示可點擊的「加入購物車」按鈕
```

### Scenario: 商品庫存歸零時禁用購買按鈕
```
Given 商品 B 的庫存數量為 0
When 使用者進入商品 B 的頁面
Then 頁面應顯示禁用狀態的購買按鈕（不可點擊）
And 應顯示「庫存不足」提示文字
```

### Scenario: 加入購物車時庫存已被他人購買完
```
Given 使用者在商品頁面，顯示庫存尚有 1 件
When 使用者點擊「加入購物車」
And 此時庫存已被其他人購買而歸零
Then 系統應顯示「商品已售完」錯誤提示
And 購買按鈕應更新為禁用狀態
```
