# 功能流程：訂單狀態通知信修正

## 功能概述

修正訂單狀態更新後，通知信未正確觸發寄送給買家的問題，重新設計通知觸發流程以確保可靠性。

## 流程說明

### 觸發條件

當後台管理員或系統將訂單狀態更新為以下任一狀態時，應觸發通知信：

- 已確認（confirmed）
- 已出貨（shipped）
- 已完成（completed）
- 已取消（cancelled）

### 修正後的正確行為

**1. 狀態更新**

- 系統接收訂單狀態更新請求
- 驗證狀態轉移合法性（如：不可從已完成回到已出貨）
- 若狀態轉移合法，持久化新狀態至資料庫

**2. 通知觸發**

- 狀態更新成功後，立即觸發通知事件
- 通知服務接收事件，查詢買家 email 與訂單資訊
- 依據新狀態選擇對應的 email 模板

**3. 寄送通知信**

- 組合 email 內容（訂單編號、商品清單、新狀態說明）
- 呼叫 email 服務寄送
- 若寄送失敗，記錄錯誤日誌並支援重試機制

**4. 邊界條件**

- 若買家 email 不存在或格式錯誤 → 記錄錯誤，不阻斷狀態更新流程
- 若 email 服務暫時不可用 → 加入重試佇列，最多重試 3 次
- 同一訂單同一狀態不重複寄送（防冪等性）
