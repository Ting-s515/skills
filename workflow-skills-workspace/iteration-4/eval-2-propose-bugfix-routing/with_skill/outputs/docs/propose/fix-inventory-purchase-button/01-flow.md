# 功能流程：庫存歸零時停用購買按鈕修正

## 功能概述

修正商品庫存歸零後，前台「可購買」按鈕未根據庫存狀態動態更新的問題，確保庫存歸零時按鈕狀態正確反映為不可購買。

## 流程說明

### 修正後的正確行為

**1. 商品頁面載入**

- 前台載入商品詳情頁時，取得商品庫存數量
- 若庫存 > 0 → 顯示啟用的「加入購物車」按鈕
- 若庫存 = 0 → 顯示禁用的按鈕，並標示「已售完」或「補貨中」

**2. 庫存狀態動態更新**

- 當其他使用者購買導致庫存變化時，前台應能感知最新庫存狀態
- 支援兩種更新機制（擇一或組合）：
  - 定期輪詢（polling）：每隔固定時間重新查詢庫存
  - 即時推送（WebSocket / SSE）：庫存變更時由後端主動通知前台

**3. 按鈕狀態切換**

- 庫存從有貨變為歸零 → 按鈕即時禁用，防止點擊
- 庫存從歸零恢復有貨 → 按鈕重新啟用

**4. 後端 API 防衛**

- 即使前台按鈕未禁用，後端在接收購買請求時仍需驗證庫存是否充足
- 若庫存不足 → 回傳明確錯誤訊息（如 HTTP 409），前台顯示庫存不足提示

**5. 邊界條件**

- 商品在購物車中且庫存後來歸零 → 結帳時由後端攔截，前台顯示錯誤提示
- 多使用者同時購買最後一件商品 → 後端採用樂觀鎖或原子操作防止超賣
