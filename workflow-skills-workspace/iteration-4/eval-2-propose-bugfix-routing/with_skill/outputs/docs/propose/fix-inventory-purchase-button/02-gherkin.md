# 驗收條件：庫存歸零時停用購買按鈕修正

## Feature: 商品購買按鈕依庫存狀態動態顯示

### Scenario: 商品有庫存時，購買按鈕為啟用狀態

```gherkin
Given 商品「藍芽耳機」的庫存數量為 5
When 使用者進入該商品的前台詳情頁
Then 應顯示啟用的「加入購物車」按鈕
And 按鈕可被點擊
```

### Scenario: 商品庫存歸零時，購買按鈕為禁用狀態

```gherkin
Given 商品「藍芽耳機」的庫存數量為 0
When 使用者進入該商品的前台詳情頁
Then 應顯示禁用的「加入購物車」按鈕
And 按鈕旁應標示「已售完」字樣
And 按鈕不可被點擊
```

### Scenario: 庫存在頁面開啟後歸零，按鈕即時更新為禁用

```gherkin
Given 使用者已開啟商品「藍芽耳機」的詳情頁，庫存為 1
When 其他使用者購買了最後一件，庫存變為 0
Then 該使用者頁面上的「加入購物車」按鈕應自動變為禁用
And 應顯示「已售完」標示
```

### Scenario: 後端拒絕庫存不足的購買請求

```gherkin
Given 商品「藍芽耳機」的庫存為 0
When 使用者（透過非正常途徑）送出購買請求
Then 後端應回傳 HTTP 409 錯誤
And 回應內容應包含「庫存不足」訊息
And 資料庫庫存數量不應減少
```

### Scenario: 多使用者同時購買最後一件，僅一人成功（防超賣）

```gherkin
Given 商品「藍芽耳機」的庫存為 1
When 兩位使用者同時送出購買請求
Then 只有其中一位使用者購買成功
And 另一位使用者應收到「庫存不足」的錯誤回應
And 資料庫庫存數量應精確扣為 0，不低於 0
```

### Scenario: 庫存從零恢復有貨時，按鈕重新啟用

```gherkin
Given 商品「藍芽耳機」原本庫存為 0，按鈕顯示為禁用
When 管理員補貨，庫存更新為 10
Then 前台的「加入購物車」按鈕應重新變為啟用
And 「已售完」標示應消失
```
