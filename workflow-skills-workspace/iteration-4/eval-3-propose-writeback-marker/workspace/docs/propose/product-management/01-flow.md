# 功能流程：商品管理

## 功能概述

管理員可在後台新增商品、瀏覽商品列表（支援名稱篩選與價格排序），並對每筆商品進行編輯或下架操作。

## 流程說明

### 1. 查看商品列表

1. 管理員進入「商品管理」頁面
2. 系統載入商品列表（預設顯示全部上架商品）
3. 列表顯示欄位：名稱、價格、庫存數量、狀態

### 2. 篩選與排序商品

1. 管理員在搜尋欄輸入商品名稱關鍵字
   - IF 有輸入 → 列表過濾顯示名稱包含關鍵字的商品
   - IF 無輸入 → 顯示全部商品
2. 管理員點選「依價格排序」按鈕
   - 第一次點選 → 由低到高排序
   - 再次點選 → 由高到低排序

### 3. 新增商品

1. 管理員點擊「新增商品」按鈕
2. 系統顯示新增商品表單
3. 管理員填寫欄位：名稱（必填）、價格（必填，正整數）、庫存數量（必填，非負整數）、商品描述（選填）
4. 管理員送出表單
   - IF 必填欄位未填 → 顯示欄位驗證錯誤，不送出
   - IF 欄位格式錯誤 → 顯示格式錯誤提示，不送出
   - IF 驗證通過 → 送出 API 新增商品
     - IF API 成功 → 關閉表單，列表加入新商品，顯示成功訊息
     - IF API 失敗 → 顯示錯誤訊息，表單不關閉

### 4. 編輯商品

1. 管理員在列表點擊某筆商品的「編輯」按鈕
2. 系統顯示編輯表單，預填現有資料
3. 管理員修改欄位後送出
   - 驗證與 API 流程同新增商品
   - IF API 成功 → 列表更新商品資料，顯示成功訊息

### 5. 下架商品

1. 管理員在列表點擊某筆商品的「下架」按鈕
2. 系統顯示確認對話框
   - IF 管理員確認 → 送出 API 請求將商品狀態改為「下架」
     - IF API 成功 → 列表更新狀態，顯示成功訊息
     - IF API 失敗 → 顯示錯誤訊息，狀態不變
   - IF 管理員取消 → 關閉對話框，不做任何變更
