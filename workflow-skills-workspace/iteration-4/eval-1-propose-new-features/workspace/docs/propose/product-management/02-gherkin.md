# 驗收條件：商品管理

## Feature: 商品列表

```gherkin
Feature: 商品列表查詢
  作為管理員
  我希望能查看所有商品列表並依名稱篩選與依價格排序
  以便管理商品上下架與庫存狀況

  Scenario: 進入商品管理頁面，顯示所有商品
    Given 管理員已登入後台
    When 管理員點擊側邊欄「商品管理」
    Then 系統顯示所有商品列表
    And 列表包含名稱、價格、庫存數量、狀態欄位
    And 列表依建立時間降序排列

  Scenario: 依名稱關鍵字篩選商品 — 有結果
    Given 管理員在商品管理頁面
    When 管理員在篩選框輸入名稱關鍵字「手機」
    Then 系統顯示所有名稱包含「手機」的商品（不區分大小寫）

  Scenario: 依名稱關鍵字篩選商品 — 無結果
    Given 管理員在商品管理頁面
    When 管理員在篩選框輸入不存在的商品名稱
    Then 系統顯示「查無符合商品」提示
    And 列表為空

  Scenario: 清空篩選框還原列表
    Given 管理員已在篩選框輸入關鍵字並看到篩選結果
    When 管理員清空篩選框
    Then 系統還原顯示所有商品列表

  Scenario: 依價格升序排序
    Given 管理員在商品管理頁面
    When 管理員第一次點擊價格欄位標頭
    Then 系統依價格從低到高重新排列商品列表

  Scenario: 依價格降序排序
    Given 管理員已點擊一次價格欄位標頭（目前為升序）
    When 管理員第二次點擊價格欄位標頭
    Then 系統依價格從高到低重新排列商品列表

  Scenario: 篩選與排序同時套用
    Given 管理員已依名稱篩選出部分商品
    When 管理員點擊價格欄位標頭進行排序
    Then 系統對篩選後的結果依價格排序
```

## Feature: 新增商品

```gherkin
Feature: 新增商品
  作為管理員
  我希望能新增商品至系統
  以便上架新產品供消費者購買

  Scenario: 成功新增商品
    Given 管理員在商品管理頁面
    When 管理員點擊「新增商品」按鈕
    And 管理員填寫名稱「藍牙耳機」、價格「1200」、庫存數量「50」、商品描述「高音質藍牙耳機」
    And 管理員點擊送出
    Then 系統呼叫 API 建立商品
    And 表單關閉
    And 商品列表顯示新增的商品

  Scenario: 必填欄位未填寫時不允許送出
    Given 管理員在新增商品表單
    When 管理員未填寫商品名稱就點擊送出
    Then 系統顯示「名稱為必填欄位」錯誤提示
    And 表單不送出

  Scenario: 價格格式錯誤
    Given 管理員在新增商品表單
    When 管理員填寫價格為「-100」或「abc」
    And 管理員點擊送出
    Then 系統顯示「價格必須為大於 0 的數字」錯誤提示
    And 表單不送出

  Scenario: 庫存數量格式錯誤
    Given 管理員在新增商品表單
    When 管理員填寫庫存數量為「-1」或小數
    And 管理員點擊送出
    Then 系統顯示「庫存數量必須為大於等於 0 的整數」錯誤提示
    And 表單不送出

  Scenario: 新增商品 API 呼叫失敗
    Given 管理員填寫完整且正確的商品資料
    When 管理員點擊送出但 API 回傳錯誤
    Then 系統顯示錯誤提示
    And 表單維持開啟狀態
```

## Feature: 編輯商品

```gherkin
Feature: 編輯商品
  作為管理員
  我希望能修改已上架商品的資訊
  以便更新商品資料

  Scenario: 成功編輯商品
    Given 管理員在商品管理列表頁
    When 管理員點擊某商品的「編輯」按鈕
    Then 系統顯示編輯表單，並預填現有商品資料
    When 管理員修改名稱後點擊送出
    Then 系統呼叫 API 更新商品資料
    And 表單關閉
    And 列表即時顯示更新後的商品資料

  Scenario: 編輯表單驗證失敗
    Given 管理員在編輯商品表單
    When 管理員將價格清空後點擊送出
    Then 系統顯示欄位錯誤提示
    And 表單不送出

  Scenario: 編輯商品 API 呼叫失敗
    Given 管理員在編輯商品表單填寫完整資料
    When 管理員點擊送出但 API 回傳錯誤
    Then 系統顯示錯誤提示
    And 表單維持開啟狀態
```

## Feature: 下架商品

```gherkin
Feature: 下架商品
  作為管理員
  我希望能將不再販售的商品下架
  以便管理商品上市狀態

  Scenario: 成功下架商品
    Given 管理員在商品管理列表頁
    And 目標商品狀態為「上架中」
    When 管理員點擊「下架」按鈕
    Then 系統顯示確認對話框「確定要下架此商品嗎？」
    When 管理員點擊確認
    Then 系統呼叫 API 執行下架
    And 商品狀態更新為「已下架」
    And 列表即時反映最新狀態

  Scenario: 管理員取消下架操作
    Given 管理員點擊「下架」按鈕後系統顯示確認對話框
    When 管理員點擊取消
    Then 對話框關閉
    And 商品狀態維持不變

  Scenario: 下架後商品在後台仍可見
    Given 某商品已被下架
    When 管理員查看商品列表
    Then 該商品仍顯示在列表中，狀態標示為「已下架」

  Scenario: 管理員權限不足時嘗試下架商品
    Given 當前管理員帳號無下架商品權限
    When 管理員點擊「下架」並確認
    Then 系統顯示「權限不足」錯誤提示
    And 商品狀態維持不變
```
