# 功能流程：使用者管理

## 功能概述

管理員可透過後台查看所有使用者列表、依 email 搜尋、查看使用者詳情，並對帳號執行停用或啟用操作。

## 流程說明

### 1. 進入使用者管理頁面

1. 管理員登入後台，點擊側邊欄「使用者管理」
2. 系統載入使用者列表（預設顯示所有使用者，依建立時間降序排列）
3. 頁面顯示：使用者列表（含 email、姓名、帳號狀態、建立時間、訂單數量欄位）

### 2. 依 email 搜尋使用者

1. 管理員在搜尋框輸入 email 關鍵字
2. 系統依輸入值即時（或點擊搜尋按鈕後）篩選使用者列表
   - IF 有符合的使用者 → 顯示篩選後列表
   - IF 無符合的使用者 → 顯示「查無符合使用者」提示
3. 管理員清空搜尋框 → 系統還原顯示所有使用者列表

### 3. 查看使用者詳情

1. 管理員點擊列表中的某一筆使用者
2. 系統導向（或彈出）使用者詳情頁
3. 詳情頁顯示：姓名、email、建立時間、訂單數量
   - IF 使用者存在 → 正常顯示詳情
   - IF 使用者不存在（已刪除）→ 顯示錯誤提示，返回列表

### 4. 停用使用者帳號

1. 管理員在列表或詳情頁點擊「停用」按鈕
2. 系統顯示確認對話框：「確定要停用此帳號嗎？」
3. 管理員確認
   - IF 確認 → 系統呼叫 API 執行停用，帳號狀態更新為「已停用」，列表即時反映狀態
   - IF 取消 → 關閉對話框，不執行任何操作
4. 已停用帳號的使用者無法登入系統

### 5. 啟用使用者帳號

1. 管理員在列表或詳情頁對「已停用」帳號點擊「啟用」按鈕
2. 系統顯示確認對話框：「確定要啟用此帳號嗎？」
3. 管理員確認
   - IF 確認 → 系統呼叫 API 執行啟用，帳號狀態更新為「啟用中」，列表即時反映狀態
   - IF 取消 → 關閉對話框，不執行任何操作

### 邊界條件

- 搜尋框支援部分比對（contains），不區分大小寫
- 停用/啟用操作需有足夠的管理員權限，否則顯示「權限不足」錯誤
- API 呼叫失敗時顯示錯誤提示，不更新前端狀態
