# 驗收條件：商品管理

```gherkin
Feature: 商品管理
  管理者可在後台新增與編輯商品，系統對欄位進行驗證，成功後即時反映於商品列表

  # ── 新增商品 ──────────────────────────────────────────

  Scenario: 成功新增商品
    Given 管理者已登入後台
    When 管理者提交名稱、售價大於 0、庫存大於等於 0 的新增請求
    Then 商品被寫入資料庫
    And 回傳新增成功的商品資料，HTTP 201

  Scenario: 新增商品 - 名稱為空
    Given 管理者已登入後台
    When 管理者提交名稱為空的新增請求
    Then 回傳驗證錯誤，訊息為「名稱為必填欄位」

  Scenario: 新增商品 - 售價為 0 或負數
    Given 管理者已登入後台
    When 管理者提交售價 ≤ 0 的新增請求
    Then 回傳驗證錯誤，訊息為「售價必須大於 0」

  Scenario: 新增商品 - 庫存為負數
    Given 管理者已登入後台
    When 管理者提交庫存 < 0 的新增請求
    Then 回傳驗證錯誤，訊息為「庫存數量必須大於等於 0」

  Scenario: 新增商品 - 庫存為 0（邊界值）
    Given 管理者已登入後台
    When 管理者提交庫存為 0 的新增請求，其餘欄位合法
    Then 商品被寫入資料庫
    And 回傳新增成功的商品資料，HTTP 201

  # ── 編輯商品 ──────────────────────────────────────────

  Scenario: 成功編輯商品
    Given 資料庫中存在 id 為 x 的商品
    When 管理者提交合法的欄位更新請求
    Then 商品資料被更新至資料庫
    And 回傳更新後的完整商品資料，HTTP 200

  Scenario: 編輯不存在的商品
    Given 資料庫中不存在 id 為 x 的商品
    When 管理者對 id x 發送編輯請求
    Then 回傳 HTTP 404，訊息為「商品不存在」

  Scenario: 編輯商品 - 更新售價為 0 或負數
    Given 資料庫中存在 id 為 x 的商品
    When 管理者提交售價 ≤ 0 的編輯請求
    Then 回傳驗證錯誤，訊息為「售價必須大於 0」

  Scenario: 編輯商品 - 更新庫存為負數
    Given 資料庫中存在 id 為 x 的商品
    When 管理者提交庫存 < 0 的編輯請求
    Then 回傳驗證錯誤，訊息為「庫存數量必須大於等於 0」

  Scenario: 編輯商品 - 更新名稱為空字串
    Given 資料庫中存在 id 為 x 的商品
    When 管理者提交名稱為空字串的編輯請求
    Then 回傳驗證錯誤，訊息為「名稱為必填欄位」
```
