# 功能流程：商品管理

## 功能概述

管理者可在後台新增與編輯商品資訊，系統對必填欄位與數值範圍進行驗證，成功儲存後即時更新商品列表。

## 流程說明

### 比對條件

| 欄位 | 規則 |
|------|------|
| `name` | 必填，不可為空字串 |
| `price` | 必填，需大於 0 |
| `stock` | 必填，需大於等於 0 |

---

### 新增商品

1. 管理者提交新增商品請求，請求 body 包含 `name`、`price`、`stock` 欄位。

2. 系統對每個欄位進行驗證：

   - **`name` 為空或未提供時：**
     回傳驗證錯誤，訊息：「名稱為必填欄位」，結束流程。

   - **`price` 未提供或 `price` ≤ 0 時：**
     回傳驗證錯誤，訊息：「售價必須大於 0」，結束流程。

   - **`stock` 未提供或 `stock` < 0 時：**
     回傳驗證錯誤，訊息：「庫存數量必須大於等於 0」，結束流程。

3. 所有欄位驗證通過後，將商品資料寫入資料庫。

4. **寫入成功：**
   回傳新增成功的商品資料（含系統產生的 `id`），HTTP 201。

5. **寫入失敗（資料庫異常）：**
   回傳系統錯誤，HTTP 500，結束流程。

---

### 編輯商品

1. 管理者提交編輯商品請求，URL 帶有目標商品 `id`，request body 包含欲更新欄位。

2. 系統依 `id` 查詢商品：

   - **商品不存在時：**
     回傳 HTTP 404，訊息：「商品不存在」，結束流程。

3. 對提交的欄位進行驗證（僅驗證有傳入的欄位）：

   - **`name` 傳入但為空字串時：**
     回傳驗證錯誤，訊息：「名稱為必填欄位」，結束流程。

   - **`price` 傳入但 ≤ 0 時：**
     回傳驗證錯誤，訊息：「售價必須大於 0」，結束流程。

   - **`stock` 傳入但 < 0 時：**
     回傳驗證錯誤，訊息：「庫存數量必須大於等於 0」，結束流程。

4. 驗證通過後，將更新後的商品資料寫入資料庫。

5. **寫入成功：**
   回傳更新後的完整商品資料，HTTP 200。

6. **寫入失敗（資料庫異常）：**
   回傳系統錯誤，HTTP 500，結束流程。

---

### 邊界條件（Out of Scope 說明）

| 情境 | 處理方式 |
|------|----------|
| `price` 為非數字型別 | 由 request 型別驗證層（DTO/schema）攔截，HTTP 400 |
| `stock` 為小數（如 1.5） | Out of Scope，目前規格未定義，預設允許但建議後續補充整數限制 |
| 管理者未登入 / 無權限 | Out of Scope，由 Auth middleware 處理，不在本功能流程內 |
| 商品名稱重複 | Out of Scope，目前規格未定義唯一性約束 |
