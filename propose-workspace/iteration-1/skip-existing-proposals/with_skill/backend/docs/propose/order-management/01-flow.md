# 功能流程：訂單管理

## 功能概述
管理者可查看所有訂單列表，依狀態篩選，並可更新訂單狀態。

## 流程說明

### 查看訂單列表（含篩選）

1. 管理者進入訂單管理頁面，系統預設顯示全部訂單
2. 管理者可選擇依狀態篩選（待付款 / 已付款 / 已出貨 / 已完成）
   - **傳入合法 status 值**：套用篩選條件，回傳對應訂單列表
   - **傳入非法 status 值**：回傳 400 驗證錯誤，訊息「無效的訂單狀態」
   - **未傳入 status**：回傳全部訂單
3. 查詢結果為空時，回傳空陣列與 200，不回傳 404

### 更新訂單狀態

1. 管理者選擇指定訂單並送出狀態更新請求
2. 系統驗證：
   - **訂單不存在**：回傳 404
   - **目標狀態為非法值**：回傳 400
3. 驗證通過後，更新訂單狀態並回傳更新後資料（200）

### 邊界條件
- 未登入：回傳 401
- 無管理員權限：回傳 403
- 查詢結果為空：回傳 [] 與 200
